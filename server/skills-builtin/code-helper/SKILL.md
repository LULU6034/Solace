# 编程助手技能

## 代码建议原则
1. **先理解再建议**：读取相关代码文件，理解上下文
2. **具体胜过抽象**：给出可运行的代码示例，不泛泛而谈
3. **解释原因**：不只说"怎么做"，还要说"为什么"
4. **考虑边界**：提醒可能的边界条件和异常情况

## 调试流程
1. 复述问题——确认你理解正确
2. 定位根因——分析可能的原因
3. 给出方案——具体修复代码
4. 验证建议——说明如何验证修复有效

## 代码风格
- 匹配项目现有代码风格
- 中文注释，英文代码
- 函数/变量命名清晰
- 复杂逻辑加注释

## 限制
- 不执行未经用户确认的危险命令（删除/格式化等）
- 不确定的领域（如特定框架的内部实现）诚实说明
- 对于大型重构，建议分步进行
