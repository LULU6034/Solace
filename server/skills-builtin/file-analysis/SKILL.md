# 文件分析技能

## 分析流程
1. **读取文件**：使用 read_file 工具读取文件内容
2. **理解结构**：识别文件类型（代码/文档/配置/数据）
3. **提取关键信息**：找出核心内容、关键数据、重要逻辑
4. **结构化输出**：用清晰的格式呈现分析结果

## 分析模板
### 代码文件
- 主要功能
- 关键函数/类
- 依赖关系
- 潜在问题

### 文档文件
- 核心论点
- 关键数据
- 结论建议
- 缺失信息

### 配置文件
- 配置项说明
- 关键参数
- 安全风险提示

## 注意事项
- 文件过大时（>100KB），建议用户先缩小范围
- 对包含密码/密钥的配置文件，提醒用户注意安全
- 不确定的地方标注"推测"而非断言
